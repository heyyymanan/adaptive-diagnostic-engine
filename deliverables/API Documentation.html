API Documentation

The Adaptive Diagnostic Engine exposes a high-performance RESTful API built with FastAPI. The architecture was specifically designed for low-latency responses, supporting real-time adaptive questioning and seamless token-optimized LLM integrations.

POST/start-session

Initializes a new diagnostic testing session. It sets up a new user tracking record in MongoDB with a baseline ability score of 0.5.

Returns: 
{
  "session_id": "string",
  "ability_score": 0.5
}
GET/next-question/{session_id}

Retrieves the next question tailored to the student's current ability score. The engine dynamically calculates the appropriate question using a custom IRT-based selection algorithm, guaranteeing the student is continually challenged at their optimal level.

POST/submit-answer

Submits a student's answer, evaluates correctness against the database, and updates the internal `ability_score` using Item Response Theory (IRT) gradient descent. This feedback loop is the core of the adaptive experience.

Request Body: 
{
  "session_id": "string",
  "question_id": "string",
  "answer": "string"
}
GET/study-plan/{session_id}

Generates a hyper-personalized actionable study plan. To ensure this endpoint runs efficiently on a free-tier setup, it passes session data through a custom Token Optimization Heuristic before interacting with the LLM API. The system intelligently switches between Google Gemini and OpenAI based on provider availability to guarantee high uptime.
